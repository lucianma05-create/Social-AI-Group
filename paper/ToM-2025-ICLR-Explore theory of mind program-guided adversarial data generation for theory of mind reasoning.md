## 一句话总结内容
> 本文提出 **EXPLORETOM**，一个面向大语言模型心智理论（Theory of Mind, ToM）推理的程序引导对抗式数据生成框架，通过领域特定语言、状态/信念追踪器和 A* 搜索自动生成多样、可靠且困难的心智理论故事与问答数据，用于更稳健地评估和提升 LLM 的社会认知推理能力。

## 一句话总结创新贡献
> 本文首次将**可执行心智状态程序建模**与**对抗式故事结构搜索**结合起来，避免仅依赖人工或 LLM 生成数据导致的简单模式、标签不可靠和数据泄漏问题，并证明 EXPLORETOM 不仅能构造让 GPT-4o、Llama-3.1-70B 等强模型显著失效的评测集，还能作为训练数据生成器显著提升模型在经典 ToM benchmark 上的泛化表现。

## 举一个例子说明这篇文章的创新点
> 传统 ToM 数据集常围绕“某人离开房间、物体被移动、再问他认为物体在哪里”这类固定模板构造，容易被模型记住表面模式。EXPLORETOM 则先用程序化动作定义故事，例如“Beth 给苹果加盐后离开，私下发消息告诉 Charles，Charles 进入厨房并移动苹果，Anne 是否知道 Charles 知道苹果被加盐？”，每个动作都会自动更新真实世界状态、人物一阶信念和二阶信念，再用 A* 搜索优先寻找最容易让目标模型答错的故事结构，最后可用 LLM 将结构补写成自然故事。

## 框架图
![framework](../image/exploretom_framework.png)

> **框架工作流描述**：  
> 1. **采样故事上下文**：使用 LLM 生成合理的人物、角色、地点、物体、容器、讨论主题和可见/不可见状态变化。  
> 2. **定义可执行动作语言**：用领域特定语言表示进入/离开房间、移动物体、更新物体状态、公开/私下交流、闲聊、秘密观察、注意力分散等动作。  
> 3. **程序化状态追踪**：每个动作执行后，自动更新世界状态、人物信念、一阶信念和二阶信念。  
> 4. **自动生成问答**：根据追踪到的状态生成 state tracking、first-order belief、second-order belief 等问题，并直接由程序给出可靠标签。  
> 5. **A* 对抗搜索困难故事结构**：以目标模型在自动问题上的低准确率作为困难度目标，同时满足用户设定的人物数、动作数、房间数等约束。  
> 6. **故事补写与数据使用**：可选用 LLM 将程序化故事结构补写成自然语言故事；结构化故事主要用于可靠评测，补写故事可用于微调训练。

## 本文挑战及已有工作不足
1. 现有 ToM benchmark 往往规模有限、场景简单、模板固定，容易高估模型的心智理论能力。
2. 很多评测数据依赖人工设计或 LLM 生成，容易存在数据泄漏、模式饱和和标签不可靠问题。
3. 经典 false-belief 测试主要覆盖少量动作，如人物进出和物体移动，难以模拟真实社交互动中的交流、信息不对称和二阶信念。
4. LLM 生成的自然故事可能引入歧义或幻觉，使得 ground-truth answer 不够稳定。
5. 仅靠随机生成或互联网文本很难获得真正需要心智理论推理的数据，大量样本可能只考察普通状态追踪。
6. 现有方法多关注 inference-time prompting 或推理策略，难以直接转化为可迁移的模型能力。

## 方法核心设计

### 1. EXPLORETOM 的任务目标
EXPLORETOM 生成包含多个人物、多种动作和不同信息可见性的故事，使不同角色可能对世界状态、他人知识或他人信念持有不同看法。模型需要回答如下问题：

- **状态追踪问题**：物体现在在哪里？某件事之前发生了什么？
- **一阶信念问题**：Anne 认为苹果在哪里？
- **二阶信念问题**：Anne 是否认为 Charles 知道苹果被加盐？
- **知识/可见性问题**：某人是否知道某个状态变化发生过？

### 2. 领域特定语言（DSL）
论文用一组可执行动作来构造故事，每个动作既改变世界状态，也改变人物的信念状态。核心动作包括：

- `a_enter` / `a_leave`：人物进入或离开房间；
- `a_moveObjContainer`：物体被移动到某个容器；
- `a_moveObjRoom`：物体被移动到另一个房间；
- `a_updateObjState`：物体状态发生变化，例如被加盐、被下毒、被覆盖巧克力；
- `a_info-private` / `a_info-public`：私下或公开传递关于世界状态的信息；
- `a_chitChat-private` / `a_chitChat-public`：私下或公开闲聊某个主题；
- `a_peek` / `a_distracted`：秘密观察或注意力分散，用于建模非对称信念更新。

### 3. 状态与信念追踪器
每个动作都有前置条件和更新规则。例如，某人进入厨房后，系统会更新“他在厨房”；如果其他人在厨房，则他们知道他进来了；如果某个状态变化被某人看到，则该人物会更新对应信念。由于标签来自程序追踪，而不是 LLM 判断，因此比纯自然语言生成问答更可靠。

### 4. 自动问题生成
系统根据世界状态和信念轨迹自动生成问题与答案。论文中特别强调“interesting question”：如果换一个被询问对象，答案会发生变化，说明该问题真正涉及心智理论，而不是普通状态追踪。例如，“Anne 是否认为 Charles 知道苹果被加盐？”可能因人物信息差而不同，因此属于更有价值的 ToM 问题。

### 5. A* 对抗式搜索
为了找到对模型最难的故事，EXPLORETOM 将故事结构搜索视为图搜索问题：节点是动作序列，边表示继续追加若干动作。A* 的代价函数由两部分组成：

- `g(s)`：目标模型在当前故事自动生成问题上的准确率，越低说明越困难；
- `h(s)`：当前故事继续扩展后满足用户约束的可能性估计，例如人物数、动作类型、房间数等。

这样，系统不是随机生成再筛选，而是主动向“模型容易失败且满足条件”的故事结构搜索。

### 6. 故事补写（Story Infilling）
程序化故事结构可以直接用于高可靠评测，也可以用 LLM 补写成自然语言故事作为训练数据。为了保证补写后仍然符合原始状态轨迹，论文采用逐步补写，并用 LLM-as-judge 检查补写文本是否保留关键状态与信念关系。

## 实验设置与主要结果

### 1. 作为评测 benchmark
论文用 EXPLORETOM 为 Llama-3.1-70B-Instruct、GPT-4o 和 Mixtral-8x7B-Instruct 生成对抗式故事结构。实验覆盖 9 类动作集合、是否包含非对称信息、不同人物数、动作数和房间数。

主要发现：

1. EXPLORETOM 能生成对强模型非常困难的 ToM 数据：
   - Llama-3.1-70B 在部分设置下准确率可低至 **0%**；
   - GPT-4o 在部分设置下准确率可低至 **9%**。
2. 随着动作数增加，故事通常更难或保持同等难度。
3. 加入秘密观察、注意力分散等非对称信息机制后，模型更容易失败。
4. A* 搜索比“过量生成 + 筛选”更有效：在相同预算下，A* 生成的数据更难，并且平均故事更短。
5. 针对某个模型生成的对抗故事，对其他模型仍然具有挑战性，说明困难性并非完全过拟合单一模型。
6. 人类标注验证显示，程序化故事结构的答案标签与人类判断有 **99%** 一致率；补写故事的一致率为 **89%**，说明自然语言补写会带来一定歧义。

### 2. 作为训练数据生成器
论文用 EXPLORETOM 生成约 **79,700** 个 `(story, question, answer)` 三元组，对 Llama-3.1-8B-Instruct 进行全量微调，得到 EXPLORETOM-8B。

主要结果：

| Benchmark | Llama-3.1 8B Instruct | EXPLORETOM-8B |
|---|---:|---:|
| ToMi | 68% | **95% (+27)** |
| Hi-ToM | 30% | **59% (+29)** |
| BigToM | 75% | **81% (+6)** |
| OpenToM (F1) | 0.39 | **0.46 (+0.07)** |
| FANToM | 0.3% | 0.2% |

此外，EXPLORETOM-8B 在更长故事、更多人物的 EXPLORETOM 数据上仍保持较高准确率，说明模型并非只记住训练模板，而是学到一定可泛化的状态/信念追踪能力。

### 3. 对 ToM 能力缺失原因的分析
论文认为 LLM 的 ToM 问题至少来自两个方面：

1. **基础状态追踪能力不稳**：许多模型不仅在二阶信念问题上失败，在普通状态追踪问题上也表现不稳定。
2. **训练数据缺少真正需要 ToM 的样本**：随机生成的故事中，大量问题并不需要区分不同角色的心理状态。论文发现，在使用 ToMi 原语随机生成的故事中，至少 **78%** 的故事不真正需要 ToM；按问题粒度看，**91%–95%** 的问题是不需要 ToM 的 uninteresting questions。

因此，提升模型 ToM 能力不能只靠更多一般文本或随机故事，而需要有意识地构造“信息不对称、信念分歧、二阶认知”等目标样本。

## 印象最深刻的点
> 这篇文章最有价值的地方在于，它不是简单让 LLM 多生成一些 ToM 问题，而是把故事背后的“世界状态—人物知识—人物信念—他人信念”显式程序化，再用 A* 搜索主动寻找模型的盲点。这样既提高了标签可靠性，也让 benchmark 可以随着模型变强而持续生成更难的数据。

## 对我们的启发
1. **程序化生成 + LLM 表达** 是一种很稳健的数据构造方式：程序负责逻辑和标签，LLM 负责语言多样性。
2. 对话/社会推理任务中，关键不是生成“看起来复杂”的文本，而是生成真正包含**信息差、状态变化和长期依赖**的结构。
3. 对抗式数据生成可以用于发现模型盲点，比静态 benchmark 更适合评估快速进化的大模型。
4. 在长程说服对话研究中，可以借鉴这种思想：显式追踪用户态度、阻抗点、策略覆盖和心理状态，再构造需要长期策略演化的样本。
5. “interesting question”的概念很有启发：数据筛选时应判断样本是否真正需要目标能力，而不是只看表面任务标签。
6. 评估时需要区分基础状态追踪能力和高阶心理推理能力，否则容易把简单记忆失败误判为 ToM 失败。

## Idea是否好想
> Idea 不算特别容易想到。核心难点在于作者没有停留在“用 LLM 生成更多 ToM 数据”，而是将心理状态更新设计成可执行 DSL，并进一步用 A* 搜索自动寻找模型失败样本。这需要同时理解心理学 false-belief 测试、程序化世界建模、搜索算法和 LLM 数据生成。

## 是否有开创性
> 具有较强开创性。它将 ToM benchmark 从静态、模板化、人工/LLM 生成，推进到“程序可控 + 标签可靠 + 对抗搜索 + 可扩展训练数据”的范式，尤其适合用于持续压力测试未来模型。

## 是否属于热点
> 属于当前大模型智能体和社会认知推理方向的热点。ToM 能力直接关系到多智能体协作、用户建模、个性化助手、对话策略、社会互动理解和长期记忆推理，也是评估 LLM 是否具备更高层次社会智能的重要方向。

> 主要关键词：Theory of Mind、false-belief reasoning、mental state tracking、belief tracking、state tracking、adversarial data generation、A* search、domain-specific language、synthetic benchmark、LLM fine-tuning。

## 与其他论文的关联（可选）
> EXPLORETOM 可视为 ToMi 和 Hi-ToM 的扩展与升级：ToMi 主要支持人物进出和物体移动，Hi-ToM 加入信息传递但交互仍受限；EXPLORETOM 支持更多动作、信息不对称和更复杂的故事结构。它也与 AutoBencher 等自动 benchmark 生成工作相关，但区别在于 EXPLORETOM 不是单纯文本过生成，而是先在可执行叙事空间中搜索困难结构，再进行自然语言补写。

## 还有哪些不足的地方（未来工作）
1. 当前故事仍受限于手工定义的动作集合，新增动作需要人工编写状态和信念更新规则，扩展成本较高。
2. EXPLORETOM 主要关注 cognitive ToM，即知识、信念和状态追踪，对情绪、动机、人格等 affective ToM 覆盖不足。
3. 故事结构虽然逻辑可靠，但自然语言补写可能引入歧义或使心理状态过于显式，影响评测稳定性。
4. 当前故事不一定是目标驱动的完整叙事，未来可以让角色行动由明确目标、计划和动机驱动。
5. 现有评估主要是问答准确率，未来可进一步扩展到真实对话、协作任务和长期交互中的 ToM 能力。
6. A* 搜索和多次 LLM 调用成本较高，未来需要更高效的搜索、缓存和难例生成策略。
